webpack简介及打包流程

发表于 2021-06-22

Webpack 是一个现代 JavaScript 应用程序的静态模块打包器（module bundler）。在前端开发中，Webpack 用于将各种资源（如 JavaScript、CSS、图片等）打包成浏览器可以直接运行的静态资源包。

构建工具

早期开发网页，可以说是”刀耕火种“，都是一个个独立的文件，需要手动引入和管理。没有所谓的“模块化”的概念。
从2009年 nodejs 的出现，前端开发开始关注模块化和代码组织。
然后前端才开始出现工程化的转变，开始使用构建工具来管理项目。

早起构建工具

  • Grunt
  • Gulp

Grunt

  • 定位： 基于nodejs的任务运行器（task runner），用于自动化重复性任务。
  • 核心思想：配置优先（configuration over code）
  • 工作方式：
    • 写 Gruntfile.js 配置文件, 用JSON 风格描述任务
    • 每个任务：读文件 -> 处理文件 -> 写文件（三段式）
    • 频繁磁盘I/O 操作，导致构建速度慢。
  • 特点：
    • 配置繁琐，JSON多
    • 生态成熟、 插件多
    • 适合老项目、简单任务

Gulp

  • 定位：基于nodejs 的流式构建工具
  • 核心思想：代码优先（code over configuration）
  • 工作方式：
    • 写 gulpfile.js, 用JS 函数描述任务
    • 基于 Node Stream (流) + pipe() 管道
    • 边读边处理、边处理边输出，只读写一次磁盘
    • 速度块、内存友好
  • 特点：
    • api 极简、代码清晰
    • 流式处理，性能好
    • 现代前端项目常用

Grunt/Gulp 是任务工具，只管压缩、合并、编译这种简单流水线，它们不处理模块依赖关系。它们是无模块化时代的产物

Webpack 出现

Webpack是干嘛的？
定位：模块打包工具（module bundler）, 用于将多个模块打包成一个或多个文件，每个文件包含所有依赖的代码。
它解决的是 模块化问题：

  • 识别 import/ require
  • 构建依赖树
  • 打包、代码分割、懒加载
  • 处理图片、字体、CSS模块化
  • 兼容老浏览器
  • 热更新、开发服务器
    他是现代前端工程化的基石。

Webpack 打包流程

  1. 初始化阶段

    1. 初始化参数(从命令行或者配置文件中获取参数)
    2. 创建compiler对象
    3. 初始化编译环境
    4. 开始编译 compiler.run()
    5. 确定入口文件

  2. 构建阶段

    1. 编译模块，从入口文件开始 loader 调用js解析器
    2. 完成模块编译，构建依赖关系图

  3. 封装阶段

    1. 合并 （根据依赖关系生成多个chunk）
    2. 优化 (TreeShaking(树摇), CodeSplitting(代码分割), Minification(压缩), scope hoisting(作用提升) )
    3. 输出资源 (将所有chunk合并到指定目录)
