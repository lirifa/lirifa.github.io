中间人攻击

发表于 2022-04-20
中间人攻击是一种经典的网络攻击方式：攻击者悄悄插入通信双方之间，截获、篡改、伪造双方数据，而通信双方还以为是在直接安全对话。

中间人攻击（MITM）详细讲解

中间人攻击（Man-in-the-Middle Attack，简称 MITM），是一种经典的网络攻击方式：
攻击者悄悄插入通信双方之间，截获、篡改、伪造双方数据，而通信双方还以为是在直接安全对话。

简单比喻：
A 想给 B 写信，结果信被中间人 M 截走。M 拆开看内容、甚至改内容，再重新封好发给 B；B 回信也一样被 M 截获。A 和 B 都不知道中间有个人在偷看、篡改。

一、核心原理

1. 劫持通信链路

攻击者控制双方之间的网络通道，让流量经过自己。

2. 伪装身份

对 A 冒充 B，对 B 冒充 A。

3. 监听 / 篡改 / 转发

可以：

  • 只窃听（被动 MITM）
  • 篡改内容（转账金额、网页内容）
  • 插入恶意代码、钓鱼链接
  • 完全替换通信内容

满足两个关键条件就能实施 MITM：

  • 能获取双方的网络流量
  • 能骗过双方的身份验证

二、常见实现方式

1. ARP 欺骗（局域网最常见）

  • 同一 WiFi / 局域网内
  • 攻击者伪造 ARP 数据包，让同网段设备误以为 “攻击者的 MAC 就是网关 MAC”
  • 所有人上网流量都先经过攻击者电脑

2. DNS 劫持 / DNS 污染

  • 把域名解析指向攻击者服务器
  • 你访问 baidu.com，实际打开的是攻击者伪造页面

3. 恶意热点 / 钓鱼 WiFi

  • 公共场所免费 WiFi 由攻击者搭建
  • 你连上后，所有 HTTP 明文流量全被看光

4. SSL 剥离（降级攻击）

  • 强制把 HTTPS 降级为 HTTP
  • 让通信变回明文，方便截获

5. 证书伪造

  • 攻击者伪造网站 SSL 证书
  • 如果设备信任了伪造证书，HTTPS 加密形同虚设

三、典型攻击场景

1. 账号密码窃取

你在公共 WiFi 登录网页 / APP，密码通过 HTTP 明文传输，攻击者直接抓包拿到。

2. 篡改网页内容

  • 把银行转账金额改大
  • 把下载链接替换成木马
  • 插入广告、恶意脚本

3. 会话劫持（Cookie 劫持）

截获你的登录 Cookie，攻击者直接用你的身份登录，无需密码。

4. 钓鱼与伪造站点

你以为在登录官网，其实是攻击者搭建的高仿页面，输入即泄露。

四、HTTP 与 HTTPS 对 MITM 的区别

HTTP（明文）

  • 极易被 MITM
  • 账号、密码、内容全裸奔
  • 抓包工具可直接查看

HTTPS（加密）

  • 理论上可以抵御普通 MITM
  • 数据加密传输，即使截获也无法解密
  • 依赖 SSL/TLS 证书验证身份

但HTTPS并非绝对安全：

  • 设备信任了伪造根证书
  • 浏览器忽略证书错误
  • 企业 / 网关部署合法 SSL 代理（也算广义 MITM）

五、如何防范中间人攻击

  1. 优先使用 HTTPS，不要点 “继续访问不安全网站”
  2. 公共 WiFi 下：
    • 不登录网银、支付、重要账号
    • 不随机输入敏感信息
  3. 开启 VPN 加密通道
  4. 关闭自动连接未知 WiFi
  5. 不随意安装不明根证书
  6. 重要网站手动核对域名，不点击可疑链接
  7. 开启网站 HSTS 强制 HTTPS

六、总结

中间人攻击就是攻击者 “夹” 在你和目标之间，冒充双方偷听、篡改通信内容。
它不破坏系统，不搞破坏，只偷数据、改数据、骗数据，是网络钓鱼、信息泄露的重要手段。
