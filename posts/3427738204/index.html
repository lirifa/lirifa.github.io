webpack构建优化

发表于 2021-07-08

当项目越来越复杂时，会面临着构建速度慢和构建出来的文件体积大的问题。webapck构建优化对于大项目是必须要考虑的一件事，下面我们就从速度和体积两方面来探讨构建优化的策略。

构建速度优化（让打包更快）

1. 持久化缓存（Webpack5 最强优化）

开启后二次构建速度提升 80%～95%

开启持久化缓存后，发生了什么?

第一次构建（全量编译）

  • 正常编译所有文件
  • 自动把下面内容存在 node_modules/.cache/webpack
    • 每个文件的转译结果（babel 编译后的代码）
    • 依赖关系图
    • 哈希值
    • chunk 组成

第二次构建（巨快）

Webpack 只做 3 件事：

  1. 对比文件哈希
    没变的文件 → 直接读缓存
    变了的文件 → 只重新编译这一个

  2. 直接读取缓存的转译结果
    不再跑 loader
    不再解析
    不再分析依赖

  3. 快速重新组合

2. 多线程构建（thread-loader）

压缩、转译 JS/CSS 非常吃 CPU，多线程能快 2～5 倍

3. 缩小文件搜索范围

减少 Webpack 找文件的时间

4. 忽略不需要的文件

5. 开发环境关闭不必要优化

打包体积优化（让产物更小）

1. 代码分割（自动拆包）

Webpack5 开箱即用

2. CSS 抽离 & 压缩

3. JS 压缩（esbuild 比 terser 快 10 倍）

4. 图片压缩 & 转 Base64

5. Tree Shaking（删除无用代码）

必须满足：

  1. 使用 ES6 模块（import/export）
  2. 生产环境 mode: ‘production’

6. 移除 console.log

优化监控

优化不是瞎猜的，一定要先监控、再优化，否则你根本不知道：

  • 哪个包体积最大
  • 哪个依赖拖慢了速度
  • 哪些代码是多余的
