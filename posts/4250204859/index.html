UI覆盖攻击

发表于 2022-04-21
UI覆盖攻击是一种利用视觉欺骗的 Web 攻击手段，核心是让用户在不知情的情况下，点击了隐藏在透明层下的敏感操作按钮。

点击劫持（Clickjacking，又称 UI 覆盖攻击） 是一种利用视觉欺骗的 Web 攻击手段，核心是让用户在不知情的情况下，点击了隐藏在透明层下的敏感操作按钮。

一、攻击原理

攻击者利用 HTML <iframe> 与 CSS 层叠 / 透明度 / 定位，将目标网站（如银行、社交账号）嵌入完全透明的框架中，并精准覆盖在诱饵页面（如 “免费领取” 按钮）之上。

  • 用户看到并点击诱饵按钮
  • 实际点击的是透明 iframe 内的转账、授权、删除等敏感按钮
  • 浏览器同源策略允许跨域嵌入 iframe，但无法阻止点击事件穿透，这是攻击成立的技术基础

二、典型攻击场景

  • 金融欺诈：诱导点击 “领取红包”，实际触发银行转账
  • 权限窃取：伪装成 “关注” 按钮，实际授权第三方获取用户隐私
  • 账号破坏：伪装成 “播放视频”，实际执行 “删除所有邮件”
  • 社交滥用：Likejacking，诱导点赞 / 分享，扩散恶意内容

三、防御方案（优先级从高到低）

  1. 服务端响应头（最可靠，推荐）
    (1) X-Frame-Options（兼容性好）

  • 用户教育：警惕陌生页面的诱导点击，查看浏览器状态栏链接确认
